# 기상청(KMA) 날씨 연동 반영 가이드

이번에 추가/변경된 파일은 3개예요.
- `api/kma.js` — **새 파일**. 기상청 API를 중계하는 서버리스 함수 (좌표 변환 포함)
- `index.html` — 기상청 데이터를 가져와서 Open-Meteo 데이터와 합치도록 수정
- `sw.js` — 캐시 버전 갱신 (v7)

## 1. GitHub에 파일 반영하기

**api 폴더 새로 만들기**
1. 저장소(`ParkHomeOfficeWeather`) 메인 화면에서 **Add file → Create new file**
2. 파일명 입력란에 `api/kma.js` 라고 입력 (슬래시를 입력하면 GitHub가 자동으로 `api` 폴더를 만들어줘요)
3. 받은 `kma.js` 내용 붙여넣기
4. **Commit new file**

**index.html, sw.js 교체**
- 기존과 같은 방식으로 두 파일 내용을 새 버전으로 교체 후 Commit

## 2. Vercel에 API 키 등록하기 (가장 중요한 단계)

API 키는 GitHub 코드에는 절대 넣지 않고, Vercel에만 비공개로 등록해요.

1. Vercel 대시보드 → `parkhomeofficeweather` 프로젝트 → **Settings → Environment Variables**
2. 아래처럼 입력:
   - **Key**: `KMA_API_KEY`
   - **Value**: 기상청에서 발급받은 인증키 (아까 채팅에 붙여넣으신 그 키)
   - **Environment**: Production, Preview, Development 전부 체크
3. **Save**

## 3. 재배포

- 환경변수를 새로 추가한 뒤에는 기존 배포에 자동 반영되지 않아요. **Deployments** 탭 → 가장 최근 배포 오른쪽 **...** 메뉴 → **Redeploy** 클릭
- (또는 GitHub에 아무 파일이나 한 번 더 커밋하면 자동으로 새 배포가 되면서 환경변수가 적용돼요)

## 4. 확인하기

배포가 끝나면 브라우저에서 직접 다음 주소로 접속해서 정상 응답이 오는지 확인할 수 있어요 (숫자는 죽전동 좌표):
```
https://parkhomeofficeweather.vercel.app/api/kma?lat=37.3253&lon=127.1082
```
- `{"nx":..., "ny":..., "hourly":[...]}` 형태의 JSON이 보이면 성공
- `{"error": "..."}` 가 보이면 아래 문제해결 참고

앱 자체는 날씨 탭을 새로고침하면 자동으로 기상청 데이터를 우선 반영해서 보여줘요 (실패해도 Open-Meteo로 자연스럽게 대체되니 화면이 깨지지 않아요).

## 참고 — 데이터가 섞이는 방식
- **오늘 ~ 모레(2~3일)**: 기상청 단기예보 값 사용 (더 정확한 국내 예보)
- **그 이후 (4~7일) + 어제(온도 비교용)**: Open-Meteo 값 그대로 사용
- 기상청 API 호출이 실패하면(키 미설정, 일시 오류 등) 자동으로 Open-Meteo만으로 동작

## 문제해결
- `"KMA_API_KEY 환경변수가 설정되지 않았습니다"` → 2번 단계 다시 확인, 저장 후 재배포 필요
- `"기상청 API 오류: ..."` → 인증키가 아직 심사/활성화 중일 수 있어요. 몇 분 뒤 다시 시도
- 격자 좌표가 이상하게 나오는 경우는 거의 없지만, 서울시청 기준 좌표(37.5665, 126.9780)로 테스트했을 때 기상청 공식 격자값(nx=60, ny=127)과 정확히 일치하는 것을 확인했어요
